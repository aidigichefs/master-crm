Overview

This project contains user APIs, AI usage logging APIs, reporting APIs, and one HTML dashboard. The public base path for all endpoints is: 

https://digichefs.in/ai-usage-api

Authentication Rules

Endpoint Group Auth Method

1. Login API

POSThttps://digichefs.in/ai-usage-api/login.php

Authenticates a user and returns role plus assigned tool mapping.

Request Parameters

NameRequiredDescription
usernameYesUser login username or email
passwordYesUser password

Example Request

curl -X POST https://digichefs.in/ai-usage-api/login.php ^
  -d "username=sajan" ^
  -d "password=9b7468caa26c"

Success Response

{
  "status": true,
  "message": "Login Successful",
  "user_id": 4,
  "username": "sajan",
  "role": "USER",
  "tool_id": 1,
  "tool_name": "ABFRL"
}

Error Responses

{
  "status": false,
  "message": "Username and password are required"
}
{
  "status": false,
  "message": "Invalid Password"
}
{
  "status": false,
  "message": "User Not Found"
}

Creates a new user and assigns the user to one tool via tool_id.

Request Parameters

NameRequiredDescription

3. User Management API

POSThttps://digichefs.in/ai-usage-api/user_management.php

Single endpoint for user create, update, delete, and list actions.

Common Parameter

NameRequiredDescription
actionYescreate, update, delete, or list
usernameRequired for create/update/deleteTarget username

Create User

curl -X POST https://digichefs.in/ai-usage-api/user_management.php ^
  -d "action=create" ^
  -d "username=vikas.t@digichefs.com" ^
  -d "password=91d5a10784f0" ^
  -d "role=USER" ^
  -d "tool_id=3"

Update User

At least one of new_username, role, or tool_id must be sent.

curl -X POST https://digichefs.in/ai-usage-api/user_management.php ^
  -d "action=update" ^
  -d "username=vikas.t@digichefs.com" ^
  -d "new_username=vikas.new@digichefs.com" ^
  -d "role=ADMIN" ^
  -d "tool_id=2"

Delete User

curl -X POST https://digichefs.in/ai-usage-api/user_management.php ^
  -d "action=delete" ^
  -d "username=vikas.new@digichefs.com"

List Users

curl -X POST https://digichefs.in/ai-usage-api/user_management.php ^
  -d "action=list"

List Response Example

{
  "status": true,
  "users": [
    {
      "id": 8,
      "username": "aman.m@digichefs.com",
      "role": "USER",
      "tool_id": 3,
      "tool_name": "Sizewise"
    }
  ]
}

4. Log Usage API

POSThttps://digichefs.in/ai-usage-api/log-usage.php

Saves AI usage log entries into ai_usage_logs.

Headers

X-API-KEY: dgcf_ai_usage_2026_x7Kp92LmQ4vN8zR1tB6sY3wE
Content-Type: application/json

JSON Body

NameRequiredDescription
tool_nameYesAllowed: ABFRL, SocialMill, Sizewise
input_tokensNoDefaults to 0
output_tokensNoDefaults to 0
total_tokensNoIf 0 or missing, computed as input + output
image_countNoDefaults to 1
statusNoDefaults to success
notesNoOptional notes

Example Request

curl -X POST https://digichefs.in/ai-usage-api/log-usage.php ^
  -H "X-API-KEY: dgcf_ai_usage_2026_x7Kp92LmQ4vN8zR1tB6sY3wE" ^
  -H "Content-Type: application/json" ^
  -d "{\"tool_name\":\"ABFRL\",\"input_tokens\":1200,\"output_tokens\":800,\"image_count\":1,\"status\":\"success\",\"notes\":\"Banner generated\"}"

7. Send Today Report API

POSThttps://digichefs.in/ai-usage-api/send-today-report.php

Sends today’s usage report by email using an external mail API.

POST JSON Body

{
  "to": ["email1@example.com", "email2@example.com"]
}

Example POST Request

curl -X POST https://digichefs.in/ai-usage-api/send-today-report.php ^
  -H "Content-Type: application/json" ^
  -d "{\"to\":[\"sajan.m@digichefs.com\",\"sajanmajrekar14@gmail.com\"]}"

Direct GET Trigger

GEThttps://digichefs.in/ai-usage-api/send-today-report.php?direct=1&key=YOUR_API_SECRET

This sends the report to the hardcoded default recipients.

Implementation Notes

  • User APIs currently use form-data or URL-encoded POST fields, not JSON.
  • log-usage.php and send-today-report.php use JSON request bodies.
  • get-summary.php and get-logs.php require X-API-KEY.
  • login.php returns tool_id and tool_name so you can segregate users after login.
  • user_management.php supports all user CRUD actions through the action parameter.
