Login API Documentation

Project: MasterCRM | Generated: July 23, 2026

Overview

login.php is a POST API used to authenticate a user and return the mapped tool information assigned through tool_id.

After successful login, the response includes the user identity, role, tool ID, and tool name so you can segregate users based on the account/tool they belong to.

Required Database Structure

ALTER TABLE user_accounts
ADD COLUMN tool_id INT(11) NULL AFTER role;

ALTER TABLE user_accounts
ADD INDEX idx_tool_id (tool_id);

Tool IDs from the current ai_tools table:

ID Tool Name
1ABFRL
2SocialMill
3Sizewise

Endpoint

POST /login.php

Request Parameters

Parameter Required Description
username Yes User login username or email.
password Yes User password.

Example Request

username=sajan
password=9b7468caa26c

cURL Example

curl -X POST https://yourdomain.com/login.php ^
  -d "username=sajan" ^
  -d "password=9b7468caa26c"

Success Response

{
  "status": true,
  "message": "Login Successful",
  "user_id": 4,
  "username": "sajan",
  "role": "USER",
  "tool_id": 1,
  "tool_name": "ABFRL"
}

Error Responses

Missing Username or Password

{
  "status": false,
  "message": "Username and password are required"
}

Invalid Password

{
  "status": false,
  "message": "Invalid Password"
}

User Not Found

{
  "status": false,
  "message": "User Not Found"
}

Database Connection Failed

{
  "status": false,
  "message": "Database connection failed"
}

Response Fields After Login

Field Description
user_idPrimary key from user_accounts.
usernameLogged-in username.
roleUser role such as USER or ADMIN.
tool_idMapped tool/account ID from user_accounts.tool_id.
tool_nameTool name from the ai_tools table.

Usage Notes

Use tool_id or tool_name from the login response in your apps and APIs to restrict or segregate data for each account.
