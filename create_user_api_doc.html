Create User API Documentation

Project: MasterCRM | Generated: July 23, 2026

Overview

create_user.php is a POST API used to create login users and map each user to a specific tool account using tool_id.

This allows login-based segregation so each user is associated with one tool from the ai_tools table.

Required Database Update

ALTER TABLE user_accounts
ADD COLUMN tool_id INT(11) NULL AFTER role;

ALTER TABLE user_accounts
ADD INDEX idx_tool_id (tool_id);

Tool IDs from the current ai_tools table:

ID Tool Name
1ABFRL
2SocialMill
3Sizewise

Endpoint

POST /create_user.php

Request Parameters

Parameter Required Description
username Yes Login username or email.
password Yes User password.
role No Allowed values: USER, ADMIN. Default: USER.
tool_id Yes Must match an active tool ID from ai_tools.

Example Request

username=aman.m@digichefs.com
password=aa3154de4e1e
role=USER
tool_id=3

cURL Example

curl -X POST https://yourdomain.com/create_user.php ^
  -d "username=aman.m@digichefs.com" ^
  -d "password=aa3154de4e1e" ^
  -d "role=USER" ^
  -d "tool_id=3"

Success Response

{
  "status": true,
  "message": "User created successfully",
  "user_id": 11,
  "username": "aman.m@digichefs.com",
  "role": "USER",
  "tool_id": 3,
  "tool_name": "Sizewise"
}

Error Responses

Missing Username or Password

{
  "status": false,
  "message": "Username and password are required"
}

Invalid Role

{
  "status": false,
  "message": "Invalid role"
}

Invalid or Inactive Tool

{
  "status": false,
  "message": "Valid active tool_id is required"
}

Duplicate User

{
  "status": false,
  "message": "User already exists"
}

Insert Failure

{
  "status": false,
  "message": "Failed to create user"
}

Usage Notes

  • Assign the correct tool_id when creating each user.
  • During login, read the user’s tool_id and use it to segregate access or data.
  • If you later need one user to access multiple tools, use a separate mapping table instead of a single tool_id column.
